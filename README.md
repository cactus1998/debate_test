## 🎯 申論的核心目標
申論不只是「演講」，而是**「建立立場」**。
* **完整性**：確保己方的核心論點在規定時間內全部呈現。
* **嚴謹性**：邏輯鏈條必須完整，不留明顯漏洞。
* **清晰度**：讓裁判在不看稿的情況下也能聽懂你的架構。

---

## 🏗️ 奧瑞岡制基本架構
在政策性辯題中，通常遵循 **需、根、解、損** 四個維度：

| 維度 | 說明 | 關鍵問句 |
| :--- | :--- | :--- |
| **需 (Necessity)** | 現況下的嚴重問題。 | 現況哪裡不好？為什麼要改？ |
| **根 (Inherency)** | 問題發生的根源。 | 為什麼現有政策無法解決問題？ |
| **解 (Solvency)** | 你的方案如何解決問題。 | 換了你的做法，問題真的會變好嗎？ |
| **損 (Side Effects)** | 方案帶來的利益（或對方的弊害）。 | 這樣做除了改善問題，還有什麼額外好處？ |

## ✍️ 撰寫三要素：圖爾敏模型 (Toulmin Model)
一個完整的論點應該包含以下三個部分：

1. **主張 (Claim)**：簡潔明瞭的結論（例如：死刑無法有效嚇阻犯罪）。
2. **證據 (Data)**：支撐主張的數據、法律條文或專家證言。
3. **推論 (Warrant)**：連結「證據」與「主張」的邏輯過程（為什麼這份數據能證明你的主張？）。

> **💡 小撇步：** 每一段論點開頭請使用「標籤句」（例如：「第一，經濟效益面...」），方便裁判抄筆記。

## ⚠️ 常見錯誤避雷
* **資料堆砌**：只報數據而沒有分析，裁判聽不懂數據代表的意義。
* **邏輯跳躍**：從 A 直接推到 Z，中間缺乏銜接。
* **語速過快**：為了塞進更多字數而像機關槍一樣說話，會導致說服力下降。
* **忽略定義**：沒有界定題目中的核心關鍵詞，導致雙方雞同鴨講。

---

## 🛠️ 撰寫建議流程
1. **拆解題目**：找出題目的關鍵字與衝突點。
2. **初步搜尋**：大量閱讀相關新聞、學術論文或法律條文。
3. **擬定架構**：決定要放哪三個核心論點。
4. **填充內容**：編寫稿件，並進行**限時朗讀**演練。
5. **修訂優化**：刪除冗贅詞彙，加強論點間的銜接感。
